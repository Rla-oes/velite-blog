# Docker란 무엇일까

애플리케이션을 신속하게 구축, 테스트 및 배포할 수 있는 소프트웨어 플랫폼이다.  
즉, 어플리케이션을 넣을 수 있는 컨테이너, 빌드나 다른 환경에 그대로 보낼 수 있게 함.

## Docker의 주요 이점

- **Portability**: Laptop, Testing Environment, Cloud Server 등 어디서나 동일하게 실행 가능
- **Scalability**: Kubernetes 통합으로 동일한 컨테이너를 여러 개 생성해 수평 확장 가능
- **Speed**: 즉시 컨테이너 실행 가능, 빠른 테스트 및 배포
- **Efficiency**: Host OS의 Kernel 위에서 작동, 가볍고 빠름 (Lightweight, Fast)
- **Consistency**: Application + Environment bundling → 어디서든 동일한 환경 재현 가능

## Docker의 주요 요소

### Docker Engine
컨테이너를 관리하는 핵심 엔진

- **Docker Daemon** (Server): 백그라운드에서 실행되며, 이미지 빌드/실행/네트워크/볼륨 관리 등을 처리
- **REST API**: 내부 명령 전달용 HTTP API (CLI 및 외부 도구가 사용)
- **CLI (Command-Line Interface)**: 사용자와 상호작용하는 터미널 도구, 내부적으로는 REST API 호출

### Dockerfile
- 이미지 생성을 위한 명령어를 나열한 텍스트 파일

### Docker Image
- Dockerfile을 빌드한 결과물
- 계층화된 읽기 전용 파일 시스템
- OS, 패키지, 코드 등 포함됨

### Docker Container
- 실행 중인 Image + 그 실행 환경
- 서로 커뮤니케이션 가능

### Docker Compose
- 여러 컨테이너를 한 번에 실행 가능하게 하는 도구
- `.yaml` or `.yml` 파일로 정의

### Docker Hub
- Docker 이미지를 저장하고 공유할 수 있는 클라우드 기반 레지스트리
